# 美食家 Phase 1 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 上线美食家 Phase 1 — 腾讯文档表格 + GitHub 开源仓库，验证社区众包本地美食优惠模式。

**Architecture:** 零代码阶段。腾讯文档多维表格作为数据层，GitHub 仓库作为项目主页和分发渠道。表格按城市组织，每个城市一张表，公开查看、微信登录编辑。

**Tech Stack:** 腾讯文档（多维表格）、GitHub（README + 文档）、Markdown

---

### Task 1: 创建腾讯文档多维表格

**产出:** 一个可直接使用的腾讯文档多维表格，包含完整字段、视图和权限配置。

- [ ] **Step 1: 创建多维表格**

在腾讯文档中新建多维表格，命名为 `美食家-本地美食优惠地图`。

- [ ] **Step 2: 配置字段**

将默认列逐一修改为以下字段：

| 字段名 | 类型 | 配置 |
|--------|------|------|
| 餐厅名称 | 文本 | 必填 |
| 所在城市 | 单选 | 选项：北京/上海/广州/深圳/成都/杭州/武汉/其他 |
| 所在区/街道 | 文本 | 如"朝阳区望京" |
| 外卖平台 | 多选 | 选项：美团外卖/饿了么/京东外卖/拼多多/抖音团购/高德团购/其他 |
| 优惠内容 | 文本 | 如"满100减40，领券中心领取" |
| 优惠类型 | 单选 | 选项：满减券/折扣券/免运费/新客券/到店团购/限时秒杀 |
| 有效期至 | 日期 | |
| 优惠截图 | 附件 | 必填 |
| 使用方式 | 文本 | 如"美团App → 首页 → 领券中心" |
| 点赞数 | 数字 | 默认值 0 |
| 举报数 | 数字 | 默认值 0 |
| 分享时间 | 创建时间 | 自动字段 |

- [ ] **Step 3: 添加状态公式列**

添加公式列 `状态`，公式逻辑（腾讯文档公式语法）：

```
IF({举报数} >= 3, "⚠️ 已隐藏",
   IF({有效期至} < TODAY(), "⏰ 已过期",
      "✅ 有效"))
```

- [ ] **Step 4: 创建默认视图**

创建三个视图：

| 视图名 | 排序 | 筛选 |
|--------|------|------|
| 🔥 24小时热门 | 点赞数 降序 | 状态 = "有效"、分享时间 ≤ 24小时 |
| 👍 高赞优惠 | 点赞数 降序 | 状态 = "有效" |
| 📋 全部 | 分享时间 降序 | 无 |

- [ ] **Step 5: 配置权限**

- 分享设置 → "任何人可查看"（免登录）
- 编辑权限 → "仅登录用户可编辑"（微信扫码即可）
- 关闭下载/复制权限（防止数据被搬运）

- [ ] **Step 6: 复制分享链接**

获取分享链接，格式类似：`https://docs.qq.com/sheet/xxx`

- [ ] **Step 7: 手动添加3条种子数据**

以你所在城市为起点，添加3条真实优惠作为种子数据，示范格式。

### Task 3: 编写 README.md

**产出:** 一份面向上帝（用户+贡献者）的 README。

- [ ] **Step 1: 编写 README.md**

```markdown
# 美食家 🍜

> 基于地理位置的、社区众包的本地美食优惠地图。

你发现了一个外卖隐藏优惠券？分享到这里，让同城的朋友们也享受到。

## 这是什么

美食家是一个**非商业、开源、社区驱动**的本地美食优惠发现工具。

- 🚫 **不是返利App** — 没有佣金链接，没有CPS分成
- 🔓 **完全透明** — 表格公开，规则公开，排名只看点赞数
- 📍 **精准到街道** — 找到你身边真正可用的优惠
- 👥 **社区互助** — 每个人分享自己发现的券，大家一起省钱

## 和返利App有什么不同

| | 返利App | 美食家 |
|---|---|---|
| 赚钱方式 | 你下单，平台拿佣金 | 不赚钱 |
| 推荐逻辑 | 谁佣金高推荐谁 | 谁点赞多谁排前面 |
| 数据 | 收集你的全部行为 | 只要你填的信息 |
| 代码 | 闭源 | 开源 |

## 怎么用

### 找优惠
1. 打开 [美食家表格](这里填腾讯文档链接)
2. 找到你的城市
3. 切换到 `🔥 24小时热门` 视图
4. 挑一个，去对应App下手

### 分享优惠
1. 打开表格链接 → 微信扫码登录
2. 新增一行
3. 填写：餐厅名、优惠内容、位置、**截图（必填）**
4. 完成！同城的人就能看到了

## 规则

- ✅ 截图必填，没截图的优惠不显示
- 👍 点赞越多排越靠前
- 🚩 举报 ≥3 条自动隐藏
- ⏰ 过期优惠自动标记

## 目前覆盖城市

- （你的初始城市）
- 想加新城市？开个 Issue 告诉我们

## 贡献

- 分享优惠：直接打开表格添加
- 加新城市：开 Issue 或直接 PR
- 代码贡献：看 [CLAUDE.md](CLAUDE.md)

## 路线图

- [x] 腾讯文档表格
- [ ] Web 展示层（更好的筛选/搜索/点赞体验）
- [ ] 更多城市

## License

Apache 2.0
```

### Task 4: 初始运营策略

**产出:** 一份运营备忘，帮助前几周冷启动。

- [ ] **Step 1: 选择初始城市**

选择你当前所在的城市作为初始城市。单一城市验证模式，不贪多。

- [ ] **Step 2: 种子内容策略**

第一周，你自己每天添加 2-3 条真实优惠，让表格不空：
- 来源：你自己用的美团/饿了么里的券、小红书刷到的、朋友圈看到的
- 每个优惠截图必附，格式规范，给后来的用户当模板

- [ ] **Step 3: 传播渠道**

| 渠道 | 做法 |
|------|------|
| 朋友圈/微信群 | 分享表格链接 + "发现一个好用的东西，大家一起来补充" |
| 小红书 | 发帖 "做了一个开源的美食优惠地图，非返利" |
| GitHub | 仓库公开可被搜索 |
| 即刻/酷安 | 技术+美食社区交叉传播 |

- [ ] **Step 4: 数据监控**

每周看一下：
- 新增条目数
- 有没有人点过赞（社区活跃信号）
- 有没有广告/垃圾内容需要清理
- 有没有新城市的需求
