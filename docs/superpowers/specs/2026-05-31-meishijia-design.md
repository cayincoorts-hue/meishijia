# 美食家 — 设计文档

**日期**: 2026-05-31
**状态**: 第一阶段设计中

## 一句话描述

基于地理位置的、社区众包的、非商业驱动的本地美食优惠发现工具。

## 用户场景

1. 用户在小红书/抖音/美团发现某餐厅有隐藏优惠券
2. 打开美食家表格，找到对应城市/区域
3. 新增一行：填写餐厅名、优惠内容、位置、上传截图
4. 同区域的用户看到后去验证
5. 有效则点赞 → 排名上升
6. 失效则举报 → 下沉/隐藏
7. 任何人在自己区域内查看高赞优惠，手动去对应App使用

## 平台

- 查看：免登录，任何人打开链接即可
- 编辑：微信扫码登录腾讯文档后即可
- 初始平台：腾讯文档（多维表格）
- 数据源：用户分享，非爬虫/API

## 表格结构

| 字段 | 类型 | 说明 |
|------|------|------|
| 餐厅名称 | 文本 | 必填 |
| 所在城市 | 文本 | 自由输入，无需预填选项 |
| 所在区/街道 | 文本 | 用于筛选 |
| 外卖平台 | 下拉多选 | 美团外卖/饿了么/京东外卖/其他 |
| 优惠内容 | 文本 | 如"满100减40" |
| 优惠类型 | 下拉单选 | 满减券/折扣券/免运费/新客券/到店团购 |
| 有效期至 | 日期 | 过期自动标记 |
| 优惠截图 | 附件/图片 | 必填 |
| 使用方式 | 文本 | 如"App首页领券中心" |
| 点赞数 | 数字 | 默认0 |
| 举报数 | 数字 | 默认0，≥3隐藏 |
| 状态 | 公式 | 有效/已过期/已隐藏 |

## 组织方式

- 按城市建文件夹
- 每个城市一张表（或按区域多张sheet）
- 默认视图：点赞降序 + 24小时内优先
- 初始城市：由用户需求驱动，逐个添加

## 防恶意机制

1. 截图必填 — 无截图的行不显示
2. 结构强制 — 关键字段不可为空
3. 点赞自然筛选 — 没人赞的广告自然沉底
4. 举报阈值 — 举报 ≥3 自动标记隐藏
5. 地理隔离 — 用户只看自己关心的区域
6. 编辑历史 — 腾讯文档原生支持，可追溯可回滚
